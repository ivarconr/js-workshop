Dynamic vs. Static Typing

Dynamic typing

  • Type checking ⇒ run-time
  • Can add code and extend objects at run-time
  • Duck typing
  • Less coupling
  • Easier to build flexible code
  • more productive?

Static typing

  • Type checking ⇒ compile-time
  • Allows many type errors to be caught early
  • More coupling

Duck typing: When I see a bird that walks like a duck and swims like a duck and quacks like a duck, I call that bird a duck. (James Whitcomb Riley)

More coupling: If we change a class-name we must also change all the places they are used

Functions: Invocations
Four invocation patterns in JavaScript
  • Method Invocation Pattern - Functions bound to object. (this = Object scope)
  • Function Invocation Pattern - Functions not bound to objects (this = global scope)
  • Constructor Invocation Pattern - New object created and this bound to that.
  • Apply/call Invocation Pattern - What do you want this to be?
    //Method invocation (what is this?)
    var someObj = {
        value: 0,
        getValue: function() {
            return this.value;
        }
    } 
    someObj.getValue();
    
    //Function invocation
    var sum = function(a,b)  {
        //What is this?
        return a + b;
    } 
    sum(1,2);

Apply and Call

Allows objects to borrow methods from other objects (by changing this)

        var obj = {
            secret: 1,
            tellSecret: function() {
                return this.secret;
            }
        };
        var obj2 = {
            secret: 42
        };
        obj.tellSecret.call(obj2, /*param1, param2, ..*/);  //42
        obj.tellSecret.apply(obj2, /*[param1, param2, ..]*/);  //42

Functions: Closure
  • A closure allows a function to access variables outside its typical scope. Such a function is said to be "closed over" its free variables.
        var counter = function() {
            var x = 0;

            var increment = function() {
                return x++;
            };
            return increment;
        };

        var c = counter();
        c();    //0
        c();    //1
        c();    //2

Object: Prototype
  • ✔ We do not have "classes" in JavaScript
  • ✔ Every object includes and inrernal referenece to another object, the prototype
  • ✔ Default prototype is object
  • ✔ Holds methods and properties that should be shared by each instance
  • ✔ Each object can choose to "override" the properties and methods. Does not affect the prototype object
  • ✔ Prototype link is only used in retrival.
  • ✔ If property does not exists in the prototype-chain --> undefined
Inheritance
  • ✔ Pseudoclassical
  • ✔ Prototypal (Differntial)
  • ✔ Functional
  • Pseudoclassical => minner om klassisk arv
  • Benytter prototype kobling
  • Komplisert i JavaScript
  • Prototypal: setter objektet i fokus, vi kan gjenbruke objekter. Vi spesifiserer forskjellen
Inheritance: functional
  • ✔ Enables privacy of fields and methods
var car = function (spec) {
    var that = {};

    that.getModel = function () {
        return spec.model;
    };
    return that;
};
var enhancedCar = function (spec) {
    var that = car(spec);

    that.getMaxSpeed = function () {
        return spec.maxSpeed;
    };
    return that;
};
var bmw =  car({ model: 'bmw'});
var opel = new enhancedCar({ model: 'opel', maxSpeed: 350});
